getCodecs

  • getCodecs(ffmpegPath?: string, searchFlag?: number): Promise<Codecs>
    • Defined in info.ts:160

    Runs ffmpeg -codecs and returns its output as Codecs. This is generally not very useful, if you need to check the compatibility for a certain encoder or decoder use getEncoders or getDecoders.

    Parameters

    • Default value ffmpegPath: string = getFFmpegPath()

      Path to the ffmpeg executable.

    • Default value searchFlag: number = -1

      Codecs which don't have this flag will be omitted.

    Returns Promise<Codecs>

    All codecs supported by ffmpeg.

getDecoders

  • getDecoders(ffmpegPath?: string): Promise<Codecs>
    • Defined in info.ts:148

    Returns all the decoders supported by ffmpeg as Codecs. This is mostly useful to check if ffmpeg supports decoding a certain codec.

    example
    const decoders = await getDecoders();
if (decoders.video.has('h264')) {
  // h264 can be used for decoding
}

    Parameters

    • Default value ffmpegPath: string = getFFmpegPath()

      Path to the ffmpeg executable.

    Returns Promise<Codecs>

getRawCodecs

  • getRawCodecs(ffmpegPath?: string): Promise<Map<string, number>>
    • Defined in info.ts:203

    Runs ffmpeg -codecs and returns its output as a map of codec names and flags. This function also returns advanced information about codecs as a bitmask, don't use it if you don't know what you are doing.

    example
    const rawCodecs = await getRawCodecs();
const flags = rawCodecs.get('h264');
if (flags & LOSSY) {
  // the codec uses lossy compression
}
if (flags & INTRA_ONLY) {
  // the codec uses only intra-frames
}

    Parameters

    • Default value ffmpegPath: string = getFFmpegPath()

      Path to the ffmpeg executable.

    Returns Promise<Map<string, number>>
